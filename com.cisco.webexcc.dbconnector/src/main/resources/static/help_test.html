Test Console - Help Guide

Overview

The Test Console allows you to execute your SQL endpoints directly from the browser. This is useful for verifying that your queries are working correctly and returning the expected data before integrating them into your Contact Center flows.

How to Use

  1. Select an Endpoint:
    • Browse the list of available endpoints in the left sidebar.
    • You can use the search box to filter by name, environment, or database.
    • Click on an endpoint to load it into the main console area.
  2. Enter Parameters:
    • If the selected query requires parameters, input fields will appear in the "Request Parameters" panel.
    • The system supports both named parameters (e.g., :customerId) and positional parameters (e.g., ?).
    • Enter valid test values for each parameter.
  3. Run Request:
    • Click the "Run Request" button.
    • The system will execute the query against the configured database.
  4. View Results:
    • The JSON response from the API will be displayed in the "Response Body" panel.
    • The status indicator will show if the request was successful (200 OK) or if an error occurred.

Troubleshooting

  • Network Error: Check if the backend server is running and accessible.
  • 500 Error: There might be an issue with the SQL query or the database connection. Check the application logs for more details.
  • Empty Response: The query executed successfully but returned no rows. Verify your test data and parameters.
