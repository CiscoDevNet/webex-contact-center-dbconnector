Database Connections - Help Guide

Overview

This page allows you to manage the database connections that your SQL endpoints will use. You can define connections for different environments (DEV, UAT, PROD) and database types.

Managing Connections

  • Add Connection: Click the "Add Connection" button to define a new database source.
  • Edit: Click the "Edit" button next to a connection to modify its details (URL, username, password, etc.).
  • Delete: Click the "Delete" button to remove a connection.
    Note: You cannot delete a connection if it is currently being used by any SQL endpoints. You must delete or reassign those endpoints first.
  • Test Connection: Click the "Test" button to verify that the application can successfully connect to the database using the provided credentials.

Connection Details

  • Name: A friendly name for the connection (e.g., "Sales DB Dev").
  • Type: The database vendor (e.g., PostgreSQL, MySQL, Oracle, SQL Server).
  • Environment: The environment this connection belongs to (DEV, UAT, or PROD).
  • JDBC URL: The connection string used to connect to the database.
