Deployment - Help Guide

Overview

The Deployment page allows you to promote a tested SQL statement from a DEV environment to UAT or PROD environments. This ensures that only verified queries are made available to your live Contact Center flows.

Important: You can only deploy DEV SQL statement(s) to UAT or PROD environments.

How to Deploy

  1. Verify Source Information:
    • Review the Statement Name, Description, and Source Environment displayed at the top of the page to ensure you are deploying the correct query.
  2. Select Production Connection:
    • Use the dropdown menu to select the target Production Database Connection.
    • Note: Only connections configured in the system will appear here. If you don't see your production database, go to the Connections page to add it.
  3. Deploy:
    • Click the Deploy button.

Warnings & Confirmation

  • Overwrite Warning: If a SQL statement with the exact same name already exists in the selected Production environment, the system will warn you.
  • Confirmation: You will be asked to confirm that you want to overwrite the existing production query. This is a safety measure to prevent accidental changes to live flows.

Best Practices

  • Always test your queries in the Test Console before deploying.
  • Ensure your Production connection credentials have the appropriate permissions (usually read-only for the application).
