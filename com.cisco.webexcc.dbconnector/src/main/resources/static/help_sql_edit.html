Add/Edit SQL Statement - Help Guide

Overview

This form allows you to define a new SQL query or modify an existing one. These queries are exposed as REST API endpoints that can be consumed by your Contact Center flows.

Form Fields

  • Name (Endpoint Identifier):
    • A unique name for your query.
    • This will be part of the API URL: /api/query/{name}.
    • Use lowercase letters, numbers, and underscores (e.g., get_customer_balance).
  • Description:
    • A brief explanation of what the query does. This helps other administrators understand its purpose.
  • Database Connection:
    • Select the database where this query should run.
    • Only connections for the current environment (e.g., DEV) are listed.
  • SQL Content:
    • The actual SQL statement to execute.
    • You can use named parameters (e.g., :customerId) or positional parameters (e.g., ?) for dynamic values.
    • Example: SELECT * FROM customers WHERE id = :customerId

Testing

Before saving, you can click the "Test SQL" button. This will:

  1. Parse your SQL for parameters.
  2. Prompt you to enter test values for those parameters.
  3. Execute the query against the selected connection.
  4. Show the results (or errors) in the console window below.
Tip: Always test your queries to ensure they return the expected data structure before using them in your flows.
